Module 7: System Design & Advanced Topics

Explore large-scale architecture, design patterns, and typical C++ interview “system design” scenarios.

Lesson 1: Design Patterns (Factory, Singleton, Observer)

Common patterns that solve recurring design challenges.

  • Implement a Factory pattern that creates different Shape objects based on a string input.
  • Write a Singleton pattern for a logger, ensuring only one instance can exist.
  • Demonstrate an Observer pattern, where multiple observers respond to state changes in a subject.

Lesson 2: High-Level System Design in C++

Interviews often test system design. Think about concurrency, scaling, data distribution, and robust error handling.

  • Sketch a high-level design for a multi-threaded server that handles client requests via sockets.
  • Discuss how you'd scale it horizontally, handle failover, and manage logs across multiple instances.

Lesson 3: Common Interview Scenarios (Producer/Consumer, Thread Pools)

Producer-consumer and thread pool designs frequently appear in interviews—understand patterns, complexities, and real-world usage.

  • Implement a basic thread pool that queues tasks (function objects) and executes them on idle worker threads.
  • Discuss how to dynamically resize the pool under heavy load or scale it down when idle.
