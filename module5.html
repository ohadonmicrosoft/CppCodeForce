Module 5: Memory Management & Debugging

Dive deep into manual vs. automatic memory management, custom allocators, and essential debugging tools like Valgrind & GDB.

Lesson 1: Manual vs. Smart Allocation

Compare raw new/delete usage to modern C++ smart pointers. Understand memory leaks, double-free errors, etc.

  • Write a snippet using raw pointers that accidentally leaks memory, then fix it via std::unique_ptr.
  • Discuss how to track allocations in a large codebase for debugging.

Lesson 2: Custom Allocators & Pool Allocation

For performance-critical or embedded systems, custom allocators can reduce fragmentation or overhead.

  • Implement a simple pool allocator that doles out fixed-size blocks.
  • Integrate it into a container class or test it with std::vector's custom allocator usage.

Lesson 3: Valgrind & GDB Basics

Understand how valgrind detects leaks or invalid memory accesses. Learn GDB for stepping, breakpoints, and stack tracing.

  • Write a short program with an intentional out-of-bounds error. Run Valgrind to identify the issue.
  • Use GDB to set breakpoints and inspect variables at runtime.
