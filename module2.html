Module 2: Data Structures & Algorithms

This module dives into fundamental data structures and classic algorithms vital for C++ interviews. Focus is on complexity analysis, practical implementation, and puzzle-like challenges.

Lesson 1: Linked Lists & Iterators

Objective: Implement singly or doubly linked lists from scratch, then create custom iterators to traverse them.

  • Create a LinkedList class supporting push_front, push_back, etc.
  • Implement a forward iterator to iterate over your nodes.
  • Explain complexity for insertion/removal in different positions.

Lesson 2: Sorting & Searching

Explore QuickSort, MergeSort, HeapSort, and binary search. Understand average/worst-case complexities and when each approach is favored.

  • Implement quickSort using a random pivot for better average complexity.
  • Write a mergeSort and compare it to std::sort results for large arrays.
  • Use binary search (either custom or std::binary_search) on a sorted array of integers.

Lesson 3: Graphs & BFS/DFS

Implement graph representations (adjacency list) and traversal algorithms like BFS & DFS. Understand how to detect cycles or find shortest paths.

  • Create a Graph class with addEdge(int u, int v) for undirected or directed edges.
  • Implement BFS to find shortest distance from a start node to others.
  • Use DFS to detect cycles in a directed graph.

Lesson 4: Common Algorithmic Puzzles

Typical problems like two-sum, sliding window maximum, and more appear frequently in interviews.

  • Two-Sum: Return indices of two numbers that add to a target.
  • Sliding Window Max: For each window of size K in an array, output the maximum.
  • Analyze time complexities of naive vs. optimized approaches.
